# Claude Code Plugins

## Install

```bash
/plugin marketplace add kierr/claude
```

Alternatively, add via URL:

```bash
/plugin marketplace add https://github.com/kierr/claude
```

## Available Plugins

### docs

Access official Claude Code documentation via slash commands.

**Install:**

```bash
/plugin install docs@kierr
```

### memory

Track tool usage and extract session memories at configurable thresholds.

**Install:**

```bash
/plugin install memory@kierr
```

### trunk

Advanced Trunk integration for automated code quality and workflow automation.

**Install:**

```bash
/plugin install trunk@kierr
```

### fs-mv

Configurable file movement plugin that automatically organizes files based on patterns and project structure.

**Install:**

```bash
/plugin install fs-mv@kierr
```

### rules

Intelligent rule interpolation system with Cursor-style MDC support, pattern matching, and security enforcement. Provides context-aware assistance with enterprise-grade performance optimization.

**Install:**

```bash
/plugin install rules@kierr
```

## Repository Structure

This is a minimal telephone book that points to independent plugin repositories:

- **claude-docs**: <https://github.com/kierr/claude-docs>
- **claude-memory**: <https://github.com/kierr/claude-memory>
- **claude-trunk**: <https://github.com/kierr/claude-trunk>
- **claude-fs-mv**: <https://github.com/kierr/claude-fs-mv>
- **claude-rules**: <https://github.com/kierr/claude-rules>

Each plugin is maintained as a separate repository with its own version control and release cycle.
